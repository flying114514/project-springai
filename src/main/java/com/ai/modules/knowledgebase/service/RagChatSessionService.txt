## 🎯 完整示例：用户与 AI 的流式对话

### 场景描述
用户在 RAG 聊天系统中提问："什么是 Spring Boot？"，系统需要流式返回 AI 答案。

---

### 📝 第一步：Controller 层调用示例

```java
@RestController
@RequestMapping("/api/chat")
@RequiredArgsConstructor
public class RagChatController {

    private final RagChatSessionService chatSessionService;

    /**
     * 流式聊天接口（SSE）
     */
    @PostMapping(value = "/{sessionId}/stream", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public SseEmitter streamChat(
        @PathVariable Long sessionId,
        @RequestBody RagChatDTO.SendMessageRequest request
    ) {
        SseEmitter emitter = new SseEmitter();

        try {
            // 1️⃣ 准备消息（保存用户问题，创建 AI 占位符）
            Long aiMessageId = chatSessionService.prepareStreamMessage(
                sessionId,
                request.question()
            );

            // 2️⃣ 获取流式回答
            Flux<String> answerFlux = chatSessionService.getStreamAnswer(
                sessionId,
                request.question()
            );

            // 用于累积完整答案
            StringBuilder fullAnswer = new StringBuilder();

            // 3️⃣ 订阅流并逐步发送给前端
            answerFlux.subscribe(
                // ✅ 收到数据块时
                chunk -> {
                    fullAnswer.append(chunk);
                    try {
                        // 发送 SSE 事件给前端
                        emitter.send(SseEmitter.event()
                            .name("message")
                            .data(chunk));
                        System.out.println("发送片段：" + chunk);
                    } catch (IOException e) {
                        emitter.completeWithError(e);
                    }
                },

                // ❌ 发生错误时
                error -> {
                    log.error("流式响应出错", error);
                    emitter.completeWithError(error);
                },

                // ✅ 流式响应完成时
                () -> {
                    String completeContent = fullAnswer.toString();

                    // 4️⃣ 更新数据库中的 AI 消息
                    chatSessionService.completeStreamMessage(
                        aiMessageId,
                        completeContent
                    );

                    // 发送完成信号
                    emitter.send(SseEmitter.event()
                        .name("complete")
                        .data("[DONE]"));
                    emitter.complete();

                    log.info("流式响应完成，总长度：{}", completeContent.length());
                }
            );

        } catch (Exception e) {
            emitter.completeWithError(e);
        }

        return emitter;
    }
}
```

### 🔍 关键设计点解析

| 步骤 | 方法 | 作用 | 为什么要这样做？ |
|------|------|------|-----------------|
| **1** | `prepareStreamMessage()` | 立即保存用户消息和 AI 占位符 | ✅ 防止网络中断导致数据丢失<br>✅ 提前锁定消息顺序 |
| **2** | `getStreamAnswer()` | 获取 `Flux<String>` 流 | ✅ 响应式编程，支持背压<br>✅ 逐段生成内容 |
| **3** | `emitter.send(chunk)` | 实时推送给前端 | ✅ 用户体验好（打字机效果）<br>✅ 减少首字延迟 |
| **4** | `completeStreamMessage()` | 填充完整内容并标记完成 | ✅ 确保数据完整性<br>✅ 方便后续查询和统计 |

### 💡 如果出现问题会怎样？

#### 场景 1：流式生成中途失败
```java
// 假设在第 3 个片段后网络中断
// 数据库中消息 ID=4 的状态：
| id | content | completed |
|----|---------|-----------|
| 4  | "Spring Boot 是一个..." | false | ← 仍然是未完成状态
```


**好处：**
- 可以重试或继续生成
- 管理员可以看到哪些消息生成失败

#### 场景 2：用户刷新页面
```java
// 前端重新连接时会话详情：
List<RagChatMessageEntity> messages = messageRepository
    .findBySessionIdOrderByMessageOrderAsc(sessionId);

// 即使 AI 消息还在生成中，也能看到已保存的用户问题和部分 AI 内容
```

### 🎨 时序图总结

```
用户浏览器       Controller         Service           Database
    │                │                  │                  │
    │──提问请求────→│                  │                  │
    │                │─prepareStream──→│                  │
    │                │                  │─保存用户消息────→│
    │                │                  │─创建 AI 占位符───→│
    │                │←─返回 AI 消息 ID──│                  │
    │                │                  │                  │
    │                │─getStreamAnswer→│                  │
    │                │←────Flux<String>─│                  │
    │                │                  │                  │
    │◄───SSE 数据流───│                  │                  │
    │ (逐段显示)      │ (转发 Flux 数据)   │                  │
    │                │                  │                  │
    │                │ [流式完成]        │                  │
    │                │─completeStream─→│                  │
    │                │                  │─更新 AI 消息─────→│
    │                │                  │ (填充内容，标记完成)│
    │◄───完成信号─────│                  │                  │
    │                │                  │                  │
```


通过这个例子，你可以看到整个流式消息处理流程是如何环环相扣的：**先占位、再流式、最后填充**。这种设计既保证了实时性，又确保了数据的可靠性！🎯
